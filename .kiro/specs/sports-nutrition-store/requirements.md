# Требования к системе интернет-магазина спортивного питания

## Введение

Система интернет-магазина спортивного питания - это веб-приложение для продажи товаров спортивного питания с полнофункциональной системой управления базой данных. Проект выполняется студентом **Валиевым И. Б.** из группы **036-22 SMMr**. Основной акцент делается на демонстрации навыков работы с базой данных MySQL и создании качественного MVP с современным дизайном 2025 года.

## Глоссарий

- **Система_Магазина**: Веб-приложение интернет-магазина спортивного питания
- **База_Данных**: MySQL база данных для хранения всей информации системы
- **Товар**: Продукт спортивного питания (протеин, креатин, витамины и т.д.)
- **Цена**: Стоимость товара в узбекских сумах (UZS)
- **Пользователь**: Посетитель сайта, который может просматривать и покупать товары
- **Администратор**: Пользователь с правами управления товарами и заказами
- **Корзина**: Временное хранилище выбранных товаров перед оформлением заказа
- **Заказ**: Оформленная покупка с указанием товаров и данных покупателя
- **Категория**: Группировка товаров по типу (протеины, аминокислоты, витамины и т.д.)

## Требования

### Требование 1

**Пользовательская история:** Как посетитель сайта, я хочу просматривать каталог спортивного питания, чтобы найти нужные мне товары.

#### Критерии приемки

1. КОГДА пользователь заходит на главную страницу, ТО Система_Магазина ДОЛЖНА отобразить современный интерфейс с каталогом товаров
2. КОГДА пользователь выбирает категорию товаров, ТО Система_Магазина ДОЛЖНА показать все товары этой категории из База_Данных
3. КОГДА пользователь ищет товар по названию, ТО Система_Магазина ДОЛЖНА выполнить поиск в База_Данных и вернуть соответствующие результаты
4. КОГДА пользователь просматривает товар, ТО Система_Магазина ДОЛЖНА отобразить детальную информацию, загруженную из База_Данных
5. КОГДА товары отображаются, ТО Система_Магазина ДОЛЖНА показать актуальные цены в узбекских сумах и наличие из База_Данных

### Требование 2

**Пользовательская история:** Как покупатель, я хочу добавлять товары в корзину и оформлять заказы, чтобы приобрести спортивное питание.

#### Критерии приемки

1. КОГДА пользователь добавляет товар в корзину, ТО Система_Магазина ДОЛЖНА сохранить выбор в База_Данных
2. КОГДА пользователь просматривает корзину, ТО Система_Магазина ДОЛЖНА загрузить все добавленные товары из База_Данных
3. КОГДА пользователь изменяет количество товара, ТО Система_Магазина ДОЛЖНА обновить данные в База_Данных
4. КОГДА пользователь оформляет заказ, ТО Система_Магазина ДОЛЖНА создать запись заказа в База_Данных
5. КОГДА заказ создается, ТО Система_Магазина ДОЛЖНА обновить количество товаров на складе в База_Данных

### Требование 3

**Пользовательская история:** Как администратор, я хочу управлять товарами и заказами через административную панель, чтобы поддерживать актуальность каталога.

#### Критерии приемки

1. КОГДА администратор добавляет новый товар, ТО Система_Магазина ДОЛЖНА сохранить все данные товара в База_Данных
2. КОГДА администратор редактирует товар, ТО Система_Магазина ДОЛЖНА обновить соответствующую запись в База_Данных
3. КОГДА администратор удаляет товар, ТО Система_Магазина ДОЛЖНА удалить запись из База_Данных с проверкой связанных заказов
4. КОГДА администратор просматривает заказы, ТО Система_Магазина ДОЛЖНА загрузить все заказы из База_Данных с детальной информацией
5. КОГДА администратор изменяет статус заказа, ТО Система_Магазина ДОЛЖНА обновить статус в База_Данных

### Требование 4

**Пользовательская история:** Как разработчик, я хочу иметь надежную систему работы с базой данных, чтобы обеспечить целостность и производительность данных.

#### Критерии приемки

1. КОГДА выполняется любая операция с данными, ТО Система_Магазина ДОЛЖНА использовать подготовленные SQL-запросы для предотвращения SQL-инъекций
2. КОГДА происходит ошибка базы данных, ТО Система_Магазина ДОЛЖНА корректно обработать ошибку и вернуть понятное сообщение пользователю
3. КОГДА создается заказ, ТО Система_Магазина ДОЛЖНА использовать транзакции для обеспечения целостности данных
4. КОГДА система запускается, ТО Система_Магазина ДОЛЖНА автоматически создать необходимые таблицы в База_Данных если они не существуют
5. КОГДА выполняются операции с База_Данных, ТО Система_Магазина ДОЛЖНА логировать все критические операции для отладки

### Требование 5

**Пользовательская история:** Как студент Валиев И. Б. из группы 036-22 SMMr, я хочу продемонстрировать современный дизайн и UX, чтобы показать актуальные навыки веб-разработки.

#### Критерии приемки

1. КОГДА пользователь взаимодействует с интерфейсом, ТО Система_Магазина ДОЛЖНА обеспечить отзывчивый дизайн для всех устройств
2. КОГДА страницы загружаются, ТО Система_Магазина ДОЛЖНА использовать современные CSS-технологии и анимации 2025 года
3. КОГДА отображается информация об авторе, ТО Система_Магазина ДОЛЖНА четко показать "Выполнил: Валиев И. Б., группа 036-22 SMMr"
4. КОГДА пользователь навигирует по сайту, ТО Система_Магазина ДОЛЖНА обеспечить интуитивно понятный и современный пользовательский интерфейс
5. КОГДА происходят действия пользователя, ТО Система_Магазина ДОЛЖНА предоставлять мгновенную обратную связь через современные UI-элементы

### Требование 6

**Пользовательская история:** Как система, я должна обеспечивать парсинг и сериализацию данных, чтобы корректно обрабатывать информацию между клиентом и сервером.

#### Критерии приемки

1. КОГДА данные отправляются на сервер, ТО Система_Магазина ДОЛЖНА валидировать JSON-данные согласно определенной схеме
2. КОГДА данные сохраняются в База_Данных, ТО Система_Магазина ДОЛЖНА сериализовать сложные объекты в JSON-формат
3. КОГДА данные загружаются из База_Данных, ТО Система_Магазина ДОЛЖНА десериализовать JSON обратно в объекты приложения
4. КОГДА происходит парсинг пользовательского ввода, ТО Система_Магазина ДОЛЖНА корректно обрабатывать специальные символы и кодировку
5. КОГДА система обрабатывает файлы изображений товаров, ТО Система_Магазина ДОЛЖНА валидировать формат и размер файлов

### Требование 7

**Пользовательская история:** Как покупатель в Узбекистане, я хочу видеть цены в узбекских сумах и иметь удобные способы оплаты, чтобы комфортно совершать покупки.

#### Критерии приемки

1. КОГДА отображаются цены товаров, ТО Система_Магазина ДОЛЖНА показывать стоимость в узбекских сумах с правильным форматированием
2. КОГДА рассчитывается общая стоимость заказа, ТО Система_Магазина ДОЛЖНА корректно суммировать цены в UZS
3. КОГДА пользователь оформляет заказ, ТО Система_Магазина ДОЛЖНА сохранить итоговую сумму в узбекских сумах в База_Данных
4. КОГДА администратор добавляет товар, ТО Система_Магазина ДОЛЖНА принимать цену в формате узбекских сум
5. КОГДА система отображает статистику продаж, ТО Система_Магазина ДОЛЖНА показывать суммы в узбекских сумах
